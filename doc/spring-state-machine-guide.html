Simple State Machine 使用指南

一、简介

simple-state-machine 是一个轻量级的状态机库，专为订单生命周期管理设计。

🎯 零持久化依赖

状态由应用层管理，不依赖数据库

🔄 精细化重试控制

区分 BusinessError / TechnicalError

⚡ 事务最小化

事务仅包裹状态持久化

🛠️ 类型安全

基于 Kotlin 密封类和泛型

二、核心概念

2.1 ActionResult - 动作结果

Action 执行后返回的结果，支持成功、业务错误、技术错误三种类型：

ActionResult.kt
sealed class ActionResult {
    data class Success(val nextEvent: Enum<*>? = null) : ActionResult()
    
    sealed class Failure : ActionResult() {
        abstract val reason: String
        
        // 业务规则失败，不应重试
        data class BusinessError(override val reason: String) : Failure()
        
        // 技术故障，应该重试
        data class TechnicalError(
            override val reason: String,
            val cause: Throwable? = null,
        ) : Failure()
    }
    
    companion object {
        fun success(): ActionResult = Success()
        fun businessError(reason: String): ActionResult = Failure.BusinessError(reason)
        fun technicalError(reason: String, cause: Throwable? = null): ActionResult = 
            Failure.TechnicalError(reason, cause)
    }
}

2.2 StateChangeResult - 状态变更结果

状态机事件处理后的返回值，包含失败原因和重试判断：

StateChangeResult.kt
data class StateChangeResult>(
    val accepted: Boolean,
    val previousState: S,
    val newState: S,
    val event: Enum<*>?,
    val failureReason: FailureReason? = null,
) {
    enum class FailureReason {
        INVALID_TRANSITION,    // 不满足 transition 条件
        BUSINESS_ERROR,        // Action 业务失败
        TECHNICAL_ERROR,       // Action 技术故障
    }
    
    fun stateChanged(): Boolean = previousState != newState
    fun shouldRetry(): Boolean = failureReason == FailureReason.TECHNICAL_ERROR
}

三、使用示例

3.1 创建状态机

创建状态机实例
val stateMachine = StateMachine.restore(
    id = "order-123",
    currentState = OrderStatus.INIT,
    initialState = OrderStatus.INIT,
    transitionTable = transitionTable,
    listener = stateMachineListener,
)

3.2 发送事件

发送事件并处理结果
val result = stateMachine.sendEvent(OrderEvent.PR_APPROVED, mapOf(
    "orderNo" to "ORD-001",
    "market" to "DE"
))

when {
    result.accepted -> {
        // 状态变更成功
        log.info("State changed: ${result.previousState} -> ${result.newState}")
    }
    result.shouldRetry() -> {
        // 技术错误，应该重试
        log.error("Technical error, will retry")
    }
    else -> {
        // 业务错误或无效转换，不重试
        log.warn("Rejected: ${result.failureReason}")
    }
}

3.3 实现 Action

Action 实现示例
class PrApprovedAction(
    private val orderRepository: OrderRepository,
) : Action {
    
    override fun execute(context: StateContext): ActionResult {
        val orderNo = context.headers["orderNo"] as? String
        
        // 参数校验失败 → BusinessError（不重试）
        if (orderNo == null) {
            return ActionResult.businessError("Missing orderNo")
        }
        
        // 基础设施故障 → TechnicalError（应该重试）
        return try {
            val order = orderRepository.findByOrderNo(orderNo)
            if (order != null) {
                ActionResult.success()
            } else {
                // 创建新订单...
                ActionResult.success()
            }
        } catch (e: Exception) {
            ActionResult.technicalError("Database error: ${e.message}", e)
        }
    }
}

四、重试行为

场景 ActionResult StateChangeResult.failureReason 重试
Event 不匹配 transition - INVALID_TRANSITION ❌
Action 参数校验失败 BusinessError BUSINESS_ERROR ❌
Action 数据库失败 TechnicalError TECHNICAL_ERROR ✅
Action HTTP/Kafka 失败 TechnicalError TECHNICAL_ERROR ✅
最佳实践：
  • BusinessError：用于业务规则校验失败，如参数缺失、订单已存在、状态不一致等
  • TechnicalError：用于基础设施故障，如数据库连接失败、HTTP 调用超时、Kafka 发送失败等

五、事务边界

重要设计原则

State Machine 本身不管理事务。应用层负责：

  • 在 Action 外部执行耗时操作（HTTP、Kafka）
  • 在 Listener 中使用事务包裹持久化
sequenceDiagram participant T as TaskSpec participant SM as StateMachine participant A as Action participant L as Listener participant DB as Database T->>SM: sendEvent() [无事务] SM->>A: execute() [无事务] Note over A: HTTP/Kafka 调用 A-->>SM: ActionResult SM->>L: onStateChanged() [Transaction] L->>DB: UPDATE orders.status L->>DB: INSERT state_machine_history L-->>SM: 完成 SM-->>T: StateChangeResult

7.2 转换配置

DSL 配置
val transitionTable = transitionTable {
    transition {
        from(OrderStatus.INIT)
        to(OrderStatus.LOCAL_INITIALIZED)
        on(OrderEvent.PR_APPROVED)
        action(prApprovedAction)
    }
    
    transition {
        from(OrderStatus.LOCAL_INITIALIZED)
        to(OrderStatus.FACTORY_ORDER_SUBMITTED)
        // 无事件 = 自动转换
    }
}

7.3 Listener 实现

状态变更监听
class StateMachineListener(
    private val orderJpaRepository: OrderJpaRepository,
    private val transactionTemplate: TransactionTemplate,
) : StateChangedListener {
    
    override fun onStateChanged(context: StateContext) {
        transactionTemplate.executeWithoutResult {
            orderJpaRepository.updateStatusByOrderNo(
                context.machineId,
                context.targetState
            )
        }
    }
}

🔄 订单初始化状态机设计文档 (Fork/Join 并行等待版)

1. 状态机总览

本设计使用 Fork/Join 模式实现订单初始化流程，等待工厂系统返回 VOM 和 DOM 事件。

核心特性：
  • 订单初始化：PR审批通过后开始本地初始化
  • Fork/Join 模式：VOM 和 DOM 事件并行等待，两者都收到后才完成
  • COE 发送：本地初始化后发送 COE 到 Kafka
  • Deal 同步：提交工厂订单后同步状态到 Deal 服务
  • 失败处理：VOM_FAILED 事件直接导致初始化失败

2. Fork/Join 并行等待详解

并行等待流程

flowchart TB A[PR 审批通过] --> B[PR_APPROVED 事件] B --> C[INIT → LOCAL_INITIALIZED] C --> D[保存 Order 到 DB] D --> E[发送 COE 到 Kafka] E --> F[LOCAL_INITIALIZED → FACTORY_ORDER_SUBMITTED] F --> G[同步 Deal 服务 REST API] G --> H[Fork: FACTORY_ORDER_SUBMITTED] H --> I[等待 VOM] H --> J[等待 DOM] I --> K{收到 VOM} K -->|VOM| L[FIRST_VOM_RECEIVED] K -->|VOM_FAILED| M[ORDER_INITIALIZE_FAILED] J --> N{收到 DOM} N -->|DOM| O[FIRST_DOM_RECEIVED] L --> P{已有 DOM?} O --> Q{已有 VOM?} P -->|是| R[Join → ORDER_INITIALIZE_SUCCEED] Q -->|是| R P -->|否| L Q -->|否| O

等待规则：

  • FACTORY_ORDER_SUBMITTED 是 Fork 状态，等待 VOM 和 DOM 事件
  • 先收到 VOM → 进入 FIRST_VOM_RECEIVED，等待 DOM
  • 先收到 DOM → 进入 FIRST_DOM_RECEIVED，等待 VOM
  • 两者都收到 → Join 成功，进入 ORDER_INITIALIZE_SUCCEED
  • 收到 VOM_FAILED → 直接进入 ORDER_INITIALIZE_FAILED

3. 状态定义

状态 名称 说明 类型
INIT 初始状态 订单刚创建，等待 PR 审批通过 初始
LOCAL_INITIALIZED 本地已初始化 PR 审批通过，订单已保存到 DB，准备发送 COE 中间
FACTORY_ORDER_SUBMITTED 工厂订单已提交 (Fork) COE 已发送，Deal 已同步，等待 VOM/DOM Fork
FIRST_VOM_RECEIVED 已收到 VOM 等待 DOM 事件 子状态
FIRST_DOM_RECEIVED 已收到 DOM 等待 VOM 事件 子状态
ORDER_INITIALIZE_SUCCEED 初始化成功 (Join) VOM 和 DOM 都收到，初始化完成 终态
ORDER_INITIALIZE_FAILED 初始化失败 VOM_FAILED 事件导致失败 终态

8. TaskSpec 模式

项目使用 TaskSpec 模式处理异步事件，结合 db-scheduler 和 ShedLock 实现分布式任务调度。

8.1 架构

graph LR A[Kafka Event] --> B[OrderCommandService] B --> C[TaskScheduler] C --> D[(scheduled_tasks)] D --> E[OrderStateMachineTaskSpec] E --> F[StateMachineService] F --> G[状态机]

8.2 关键组件

组件 说明
TaskSpec<P> 任务规格接口，定义任务名称、最大重试次数、执行逻辑
LockingTaskSpec<P> 带分布式锁的任务规格，使用 ShedLock
OrderStateMachineTaskSpec 订单状态机任务，处理 VOM/DOM 事件
TaskScheduler 任务调度器，封装 db-scheduler 的 scheduleIfNotExists

8.3 TaskScheduler API

// 提交任务（返回 Boolean 表示是否成功调度）
val scheduled = taskScheduler.submit(
    spec = orderStateMachineTaskSpec,
    instanceId = "order-123-VOM-${UUID.randomUUID()}",
    payload = OrderEventPayload(orderId = 123L, event = OrderEvent.VOM)
)

// 取消任务
taskScheduler.cancel(
    taskName = "order-state-machine",
    instanceId = "order-123-VOM-xxx"
)

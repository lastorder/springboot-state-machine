🔄 订单状态机设计文档 (Fork/Join 并行验证版)

1. 状态机总览

本设计使用 Fork/Join 模式实现库存检查和报价的并行验证，提高订单处理效率。

核心特性：
  • 并行验证：库存检查和报价同时进行
  • Fork/Join 模式：两个验证分支并行执行，都成功后才进入下一阶段
  • 手动重试：验证失败后可手动触发重试（最多3次）
  • 超时控制：验证超时自动取消订单

2. Fork/Join 并行验证详解

并行验证流程

flowchart TB A[订单创建] --> B[SUBMIT_VALIDATION 事件] B --> C[Fork: PENDING_VALIDATION] C --> D[INVENTORY_CHECK 分支] C --> E[PRICING_CHECK 分支] D --> D1[发送 inventory.check.request] D1 --> D2{等待 Kafka 响应} D2 -->|成功| D3[INVENTORY_SUCCESS] D2 -->|失败| D4[INVENTORY_FAILED] E --> E1[发送 pricing.request] E1 --> E2{等待 Kafka 响应} E2 -->|成功| E3[PRICING_SUCCESS] E2 -->|失败| E4[PRICING_FAILED] D3 --> F[Join: 等待两个分支] E3 --> F F --> G[PENDING_CONFIRMATION] D4 --> H[CANCELLED] E4 --> H

验证规则：

  • 库存检查和报价检查同时发送 Kafka 请求
  • 两个验证都成功才能进入 PENDING_CONFIRMATION
  • 任一验证失败则订单取消
  • 验证超时（10分钟）自动取消订单
  • 支持手动重试，最多3次

3. 状态定义

状态 名称 说明 允许的操作 类型
CREATED 已创建 订单刚创建，可触发并行验证 取消、触发验证 初始
PENDING_VALIDATION 并行验证入口 (Fork) 并行验证的 Fork 状态，同时进入两个子状态 取消、重试 复合
INVENTORY_CHECK 库存检查中 并行验证分支之一：等待库存服务响应 - 子状态
PRICING_CHECK 报价检查中 并行验证分支之一：等待报价服务响应 - 子状态
PENDING_CONFIRMATION 待用户确认 (Join) 并行验证完成，等待用户确认 确认、拒绝、修改、取消 中间
PENDING_PAYMENT 待支付 用户已确认价格，等待支付 支付、修改、取消 中间
PAID 已支付 支付完成，等待支付系统确认 退款 中间
PENDING_SHIPMENT 待发货 支付已确认，等待仓库发货 取消 中间
SHIPPED 已发货 订单已发货，等待物流送达 - 中间
DELIVERED 已送达 订单已送达客户，流程正常结束 退款 终态
CANCELLED 已取消 订单已取消（验证失败/超时/用户取消） - 终态
REJECTED 已拒绝 用户拒绝了订单价格，订单结束 - 终态
REFUNDED 已退款 订单已退款 - 终态

5. Kafka Topics

Topic 方向 消息类型 触发时机 说明
inventory.check.request → 库存服务 InventoryCheckRequest SUBMIT_VALIDATION (并行发送) 发送库存检查请求
inventory.check.response ← 库存服务 InventoryCheckResponse INVENTORY_SUCCESS / INVENTORY_FAILED 库存检查结果
inventory.order.modified ← 库存服务 InventoryOrderModified INVENTORY_MODIFIED 库存服务主动修改订单
pricing.request → 报价服务 PricingRequest SUBMIT_VALIDATION (并行发送) 发送报价请求
pricing.response ← 报价服务 PricingResponse PRICING_SUCCESS / PRICING_FAILED 报价结果
payment.confirmed ← 支付服务 PaymentConfirmedEvent CONFIRM_PAYMENT 支付确认
order.shipped ← 仓库服务 OrderShippedEvent SHIP 发货通知
order.delivered ← 物流服务 OrderDeliveredEvent DELIVER 送达通知
order.refunded ← 退款服务 OrderRefundedEvent REFUND 退款通知
order.events → 外部系统 OrderStatusChangeEvent 所有状态变更 订单状态变更事件广播
